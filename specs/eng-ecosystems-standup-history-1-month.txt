#24642 Java: dependency extraction should work for Gradle multi-module projects
### Rationale
Currently dependency extraction fails for Gradle multi-module projects, or if it succeeds it produces incorrect results.
Usually when a task is requested on the command line without colon, e.g. ./gradlew test instead of ./gradlew :test Gradle will search for all tasks of that name in any subprojects and execute them. The dependencies task behaves differently though. It when executing ./gradlew dependencies --configuration runtimeClasspath it only executes for the root project. Furthermore this only works if the root project has the java plugin applied, since that plugin contributes the runtimeClasspath configuration.
